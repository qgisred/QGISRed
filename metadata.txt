about=<p>QGISRed is a freeware QGIS plugin developed to assist in building and analyzing hydraulic models of water distribution networks of any complexity, up to the level of detail required by Digital Twins. The plugin works seamlessly with QGIS, which enables the user to georeference all network elements, use geographic background layers, edit graphical and alphanumeric data, use geoprocessing tools, visualize data by layers, customize the symbology, etc.</p><p>QGISRed relies on an easy-to-use relational database of SHP and DBF files based on the EPANET data model, which has been extended to add new capabilities. Starting from the EPANET data model significantly reduces the learning curve.</p><p>In fact, the previous skills required on QGIS are minimal because QGISRed provides numerous tools to help build hydraulic models, to manage demand scenarios and to analyze the results without the need to use the QGIS environment, such as:</p><ul><li>Project manager for easy project loading and storage.</li><li>Automatic creation of new empty projects.</li><li>Import/export EPANET INP files. </li><li>Import basic network components and other auxiliary components managed by the Digital Twin (DT), such as service connections, isolation valves and sensors from Shapefiles.</li><li>Graphical editing tools for all network elements, including those of the DT (add, delete, move, edit vertices and nodes, reverse links, multiple selection, etc.).</li><li>Automatic generation of topological relationships between all the network elements when needed.</li><li>A set of tools to debug the network layout and verify its connectivity and hydraulic viability.</li><li>Default values for newly created elements.</li><li>Property editor for all components of the model and DT, including a browser.</li><li>Complete editor of analysis options and default values.</li><li>Automatic calculation of pipe lengths from the coordinates of the vertices.</li><li>Automatic interpolation of node elevations from DTM.</li><li>Automatic estimation of pipe roughness from the material and age of pipes.</li><li>Conversion of roughness coefficients between different head loss formulas and unit systems.</li><li>Managing multiple demands and quality sources as distinct layers and editing their properties, integrated with those of the nodes.</li><li>Automatic nodal demand allocation from global, sectoral or punctual consumptions.</li><li>Managing scenarios with multiple demands by category.</li><li>Automatic pattern and hydraulic efficiency allocation, either global or by sector.</li><li>Scenario builder for various model parameters by importing/exporting files</li><li>Powerful editors for curves, patterns, controls and rules.</li><li>Validation of any of the model properties against typical data entry errors.</li><li>Management of service connections and isolation valves as differentiated elements.</li><li>Loading demands to service connections from the meter readings, and moving them to the closest nodes.</li><li>Detection of isolated zones when closing the isolation valves and cancelling the demands on them.</li><li>Optional integration of service connections into the model.</li><li>Declaration of the meter sensors as distinct elements and the association of signals to them.</li><li>Reading field data for the simulation period from files and exporting to EPANET.</li><li>Automatic identification of hydraulic sectors and demand zones.</li><li>Identification of the isolated segments in the event of a fault at any point in the network and detection of the isolated demands.</li><li>Creation of thematic maps for most properties of the different network elements.</li><li>Model simulation by using the EPANET toolkit 2.3.</li><li>Visualization of simulation results by means of tables, colour-coded layers, arrows and floating or static labels.</li><li>Use of a navigation bar to view how results change over time, and check them with the status report.</li><li>View the statistical parameters for the entire simulation period for the variables associated with each element.</li><li>Automatic creation of legends and symbols for viewing data and results, which can be edited using a proper Legend Editor.</li><li>Location of any element in the network using its ID, and identify the adjacent elements.</li><li>Explore the properties of any element—whether data or results—from a permanent window.</li><li>Run queries based on one or more criteria to check against the data, the results, or both.</li><li>Show time series curves for one or more results on the same graph, whether or not they are of the same magnitude.</li></ul><p>Nevertheless, if the user decides to use the QGIS tools to edit the data, QGISRed provides powerful facilities to detect potential editing errors before simulating the model or to fill in the missing information.</p><p>QGISRed is currently available in English and Spanish, and automatically adapts to the language selected for the QGIS interface.</p><p>The development of the new QGISRed plugin is an initiative of the <b><a href="https://iiama.webs.upv.es/en/research/research-groups/hydraulic-networks-and-pressurised-systems/">REDHISP</a> Research Group</b> of the Institute of Water and Environmental Engineering (<b><a href="https://iiama.webs.upv.es/en/home/">IIAMA</a></b>) of the <b>Polytechnic University of Valencia</b>, <b><a href="http://www.upv.es/index-en.html">UPV</a></b> (Spain), led by <b>Professor <a href="https://www.researchgate.net/profile/Fernando_Martinez_Alzamora">Fernando Martínez Alzamora</a></b>. Its implementation was carried out up to 2022 by the <b><a href="https://waterpi.com/">WaterPi</a> company</b> under the direction of <b><a href="https://www.researchgate.net/profile/Nestor_Lerma">Néstor Lerma</a></b>, <b>PhD Eng</b>. Subsequently, <b><a href="https://ingeniousware.com/">Ingeniousware</a> GmbH</b> has collaborated in the development. <b>QGISRed versions after 0.17 are being funded by the <a href="https://www.iadb.org/en">Inter-American Development Bank</a> (IDB)</b> through contract C-RG-T4041-P001.</p><p>The QGISRed extension for QGIS is an update of the old GISRed application developed by our group for the ArcView 3.1 environment (<b><a href="https://www.researchgate.net/publication/336170857_Upgrade_of_the_GISRed_application_for_the_free_analysis_of_WDN_under_GIS_environment">see launching paper</a></b>). Currently, the name GISRed has been kept in relation to the libraries developed in .net, which are used by QGISRed to improve the processing time and constitute the core of the GISRed standalone application.</p><p>For more information, the user can check the official website of <b><a href="https://qgisred.upv.es/en">QGISRed</a></b> or the<b><a href="https://plugins.qgis.org/plugins/QGISRed/"> section of QGISRed</a></b> in the official QGIS plugin repository. Also a <b><a href="https://qgisred.gitbook.io/usermanual">User’s Manual</a></b> in English is available.</p><p><b>Requirements</b>: <b>QGIS 3.28 or higher</b> - <b>.Net Framework 4.8</b> - <b>Windows OS</b>.</p>

changelog=version 0.18:
      * Improvements to the Project Manager. New buttons for moving and exporting projects, and new options for renaming projects.
      * Identification of all layers managed by QGISRed using a unique ID, rather than by name, enabling work in multiple languages.
      * Revision of the map symbols, labels and warnings associated with all layers managed by QGISRed.
      * Storage of the style of all layers managed by QGISRed in .qml files at three levels: default, user level and project level.
      * Embedding symbols in .qml style files to avoid path dependencies.
      * Review of all the network layout’s graphic editing tools to ensure smoother and more robust operation. 
      * Optional specification of a background grid for creating geometric pipe layouts.
      * When creating a new pipe, if one end of it overlaps the layout of an existing pipe, it automatically splits to create a T-junction.
      * The symbol for multiple demands has been replaced by a double circle.
      * Every editing or other tool that interacts with the map now has its own icon associated with the mouse cursor, making it easier to identify the selected tool.
      * Greater integration of the Element Properties Editor, maintaining layer properties and refreshing data in all affected windows, including the attribute table.
      * Storage in a dedicated table of all measurements handled by QGISRed, specifying the units and decimal places to be displayed in the various unit systems and according to specific cases.
      * Creation of a dedicated Legend Editor to customise ranges or classes, colours and sizes of all legends handled by QGISRed, with wizards to customise legends automatically.
      * Reorganisation of the layers in the ‘Queries’ group and the storage of the corresponding SHP files in the project folders.
      * Addition of a new menu and toolbar to house all options related to querying data and results.
      * New tool to locate any element on the map using its ID, and identify its connected elements with the option to navigate through them.
      * New panel for viewing the data and results of any selected network element. Synchronisation of results with the current simulation time.
      * New dialogue box for creating thematic maps of certain properties associated with the different types of elements managed by QGISRed.
      * New panel for locating elements on the map that meet certain criteria in relation to the data or results. Synchronisation with the results for the current simulation time.
      * Improving the format in which the INP file is exported from QGISRed, similar to that exported from the EPANET Toolkit.
      * Replacement of the EPANET 2.2 calculation engine with the new version 2.3, up to the latest revision 2.3.5.
      * Addition of the Status Report to the results panel in a new tab, always accessible.
      * Reading simulation results directly from EPANET binary files for faster and more efficient navigation.
      * Export of all simulation results to a structured CSV file.
      * New option to display various statistics on the results throughout the entire simulation period.
      * New window to display the time-series curve for any parameter of an element throughout the simulation period. Option to overlay multiple curves for the same or different parameters.
      * Revision of the allocating algorithm for punctual demands in the Demand Builder, and automatic loading of both the link layer and a new layer for the demand points.
      * Revision of the tool for identifying hydraulic sectors and detecting isolated consumption points.
      * Revision of the algorithm for identifying isolated segments and the detection of isolated consumption points.
      * New Digital Twin options at the project level to transfer demands from the service connections to nodes. Classification of demands by patterns.
      * Rewriting the plugin code to avoid large files  and to improve traceability, maintainability and scalability. 
      * Compatibility of QGISRed version 0.18 with the new versions of QGIS 4.0.
      * Translation of all QGISRed dialogues, panels, messages, and layer names into Spanish.
      * Redesign of all icons managed by QGISRed with a more uniform and appealing appearance.
      * Web hosting of the provisional QGISRed manual in English and Spanish for online consultation via the collaborative platform GitBook.
      * Acknowledgement to the Inter-American Development Bank (IDB) for its financial support for all the improvements made in this version 0.18.
      * New notification system to alert users when new versions, courses, updates to the manual, etc. are available.
      * Fixed an issue when loading field data related to the decimal separator.
      * Fixed a bug preventing the cancellation of demand for isolated zones.
      * Limited the size of the ‘Description’ field when used to provide information on service connection demands loaded on each node.
      * Fixed an issue when the end of the Epanet report did not appear in certain cases. 
      * Fixed error caused when layers were open in different applications (e.g. two QGIS windows).
